swagger在laravel中的使用

2019-07-19 2019-07-24 开发小工具

开始前了解

推荐一个比较完成的文档https://learnku.com/laravel/t/7430/how-to-write-api-documents-based-on-swagger-php#b5a896
swagger主要是用于给api接口写文档的，接口主要有几个关键点需要文档对其进行解释

  1. 请求方法
  2. 请求的路径
  3. 请求的参数
  4. 返沪状态
  5. 返回参数
    另外强调一点，所有的swagger语句都要写在注释里面！

请求方法

主要是指GET、POST等请求方法在swagger中应该如何表示。
同时又因为一个模块或者说一个功能点需要多个接口，那我们就给这个接口打个标签，表示这几个接口都属于同一个模块的。

请求参数

参数主要有几种形式：

  1. query string： api/checktask?item_id=1
  2. path: api/checktask/1
  3. body: post打包传值的形式

复杂一点的请求方式

使用post或者put等方法的时候往往涉及到了对象、数组的多层嵌套进行传值。在swagger中书写起来比较麻烦一点。

返回参数

返回的参数中必定存在各种数组、对象的嵌套，我们在写swagger的时候其实可以参考复杂类型的参数传递方式，童谣都是数组包裹对象，或者对象包裹数组，本质上是一样的。

了解内容

很多时候我们的接口是需要带上一个token才能进行使用的，如果没有token就会锁住该接口，不允许访问。
